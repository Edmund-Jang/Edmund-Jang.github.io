Front-end

펼치기/접기
  팀원 중, 프론트엔드를 능숙하게 작업할 수 있는 사람이 없었기 때문에 본인이 나서서 진행을 하였습니다. (이미지 파일 제작과 디자인 및 css와 js의 수정 작업)
  Themewagon 사이트에 있는 기존 오픈 템플릿을 활용하다 보니, 다른 사람의 css와 js에 대한 분석력과 이해도가 상승하였습니다.

Back-end

25%

펼치기/접기
  프로젝트 당시, 프론트엔드 대부분의 작업을 도맡아서 진행했었기 때문에, 개인적인 욕심을 줄이고 백엔드는 본인보다 더 잘할 수 있는 팀원들에게 더 많은 작업을 배분하였습니다.
  프로젝트가 끝나고 백엔드(특히 JSP) 부분에서 큰 아쉬움이 남아, Spring Boot 공부를 시작하여 오히려 백엔드 능력을 한층 끌어올릴 수 있는 계기가 되었습니다.

Presentation

80%

펼치기/접기
  개인적으로 프레젠테이션 발표를 좋아하고, 백엔드에 대한 아쉬움과 '프로젝트 발표만큼은 누구보다 잘 할 수 있다'라는 생각에 팀원들과의 의견 조율 및 수정 작업을 거쳐, 전반적인 파워포인트 자료를 완성시키고 발표까지 진행하였습니다.

Flowchart  접기/펼치기
 

  스페이스 뮤직은 로그인 후 사용할 수 있는 사이트로 제작하였습니다.


  • 메인 페이지에서 마이 플레이 리스트와 회원정보 수정으로 이어지는 [마이메뉴]
  • 인기 100곡 듣기가 가능한 [TOP100] 페이지
  • 장르별 음악 듣기가 가능한 [장르] 페이지
  • 지니뮤직의 검색 결과로 이동시켜주는 [검색 기능]

    • DB설계도  접기/펼치기
     

      총 4개의 테이블을 사용하였습니다.


  • 유저 테이블에는 회원가입 시 입력되는 유저 정보를 저장
  • 뮤직 테이블에는 음악에 대한 정보를 저장
  • 장르 테이블은 조인을 통하여 뮤직 테이블과 연계
  • 유저뮤직조인 테이블은 각 유저의 선호하는 음악에 대한 정보를 저장하고 출력하기 위한 테이블 (= 플레이리스트를 만들기 위한 테이블)

    •   스페이스 뮤직의 DB는 너무 복잡하지 않으면서 필요한 요소들은 다 담을 수 있도록 하는 동시에 최소한의 테이블만을 사용한,

      효율성에 초점을 맞춘 DB 설계입니다.

    [회원가입 및 이메일 인증]  접기/펼치기
     

      회원가입 및 이메일 인증기능 설명


  • 프로필 사진 기능 (추후 프로필에서 변경 가능)
  • 아이디, 패스워드, 이름, 이메일, 휴대폰 기본적인 유효성 검사
  • 이메일 인증 기능

    •      (이메일 인증은 ajax를 사용하여, 사용자의 해당 이메일로 랜덤난수를 전송하고, 사용자가 입력한 난수와 일치 여부를 확인하여 인증을 하는 방식)


    [음악재생]  접기/펼치기
     

      음악 재생 기능 설명


  • 유튜브 id로 영상을 재생하는 js (function)을 사용하였습니다.
  • 유튜브 id 값을 통하여, 해당 유튜브 영상이 아닌 썸네일 사진을 보여줍니다.

    •      (위의 모든 유튜브 id 값은 Database에 등록된 값을 가져오게 합니다.)

  • 썸네일 사진 위에 css 스타일을 입혀 사용자에게 편의성을 제공하였습니다.

    •      (재생, 일시정지, 이전 곡, 다음 곡, 다시 듣기)

    [내 플레이리스트]  접기/펼치기
     

      My Library (내 플레이리스트) 기능 설명


  • 사용자가 원하는 곡을 리스트에 담을 수 있는 기능을 가지고 있습니다.
  • 사용자는 하트(좋아요) 버튼을 이용하여 리스트에 등록과 삭제를 할 수 있습니다.
  • 하트(좋아요) 버튼이 클릭하면, 활성화됨과 동시에 리스트에 담기며, 비활성화 시 리스트에서 삭제됩니다. (하트 버튼이 활성화되면, DB의 유저뮤직조인 테이블에서 해당 유저 아이디에 뮤직 아이디가 담기게 됩니다.)
  • 유저뮤직조인 테이블은 각 유저의 선호하는 음악에 대한 정보를 저장하고 출력하기 위한 테이블로 사용됩니다. (= 플레이리스트를 만들기 위한 테이블)
    • 전체 둘러보기  접기/펼치기



    프로젝트 후기

    프로젝트를 마치고...

    ◈ 아쉬웠던 부분

    • 백엔드 부분에서 참여도가 낮아 아쉬웠습니다.
    • 음악 스트리밍 외에 도전 해보고 싶은 주제가 있었으나, 난이도 조절 실패로 실행에 옮기지 못한 아쉬운 부분이 있습니다.
    • ex) 코인 or 주식 투자 성향 분석 및 추천 사이트

    ◈ 프로젝트를 통하여 배운 부분

    • 비대면으로 많은 시간을 진행하였기에, 자연스레 Github 사용법을 익히게 되었습니다.
    • 백엔드 부분(JSP)에서 느낀 아쉬움은 프로젝트가 끝난 후, Springboot 공부로 이어지게 되었습니다.
    • 협업을 위한 의사소통과 작업 배분의 중요성을 다시 한번 더 깨닫는 계기가 되었습니다.

    ◈ 총 리뷰

    • 백엔드에서 아쉬움이 남았지만, 프론트에서 원하는 만큼의 많은 작업을 해보았고,
    • 프로젝트에서 남은 아쉬움은 그대로 Springboot 공부로 이어져 한 층 더 성장할 수 있었던 프로젝트였습니다.
    • 또한, 협업에서 가장 중요한 커뮤니케이션 스킬을 한 번 더 점검하고 다질 수 있었습니다.
