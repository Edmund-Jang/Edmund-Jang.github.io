React Board는?


[실무 친화적구조]  접기/펼치기
 

  React 토이 프로젝트는 실무 친화적 구조로 구성하였습니다.


  • 라이브러리와 스타일(글로벌)을 하나의 폴더에 모아놓았습니다.
  • 컴포넌트는 각 페이지와 역할에 맞추어 나누어 놓았습니다.
  • 각 페이지의 구성요소는 units 폴더에 각각 나누어 놓았습니다.
  • 나누어진 페이지의 구성요소 또한 container, presenter, styles, (types) 부분으로 나누었습니다.

    •   이러한 구조를 적용함으로써, 유지 보수에 유리해지고(지뢰밭 방지) 제3자가 보았을 때, 구조를 이해하고 수정이 편해진다는 장점이 있습니다.

    참조: 게시판 Back-end는 React 강의를 수강한 "Codecamp" 에서 제공하는 GrapqhQL Back-end 서버를 사용하였습니다.


    [Firebase 활용]  접기/펼치기

      Firebase 사용


  • Firebase의 DB를 사용한 게시판을 별도로 제작하였습니다.
  • 일반 게시판처럼 수정, 삭제, 댓글 등의 다른 기능을 넣지는 않았습니다.

    • [Open API 적용]  접기/펼치기

      Open Api 적용


  • Open Api(깃허브)에서 제공하는 Api들 중 하나를 적용시켜 보았습니다.
  • 특별한 기능은 없으며, 강아지 사진을 랜덤으로 제공해주는 Api 입니다.

    • 전체 둘러보기  접기/펼치기



    프로젝트 후기

    프로젝트를 마치고...

    ◈ 아쉬웠던 부분

    • 시간상의 이유로 로그인 기능과 회원전용 게시판을 추가하지 못한 점이 아쉬웠습니다.
    • 구직활동을 하면서 계속 진행할 예정입니다.

    ◈ 프로젝트를 통하여 배운 부분

    • React 강의에서 강조했던, 실무에서의 폴더 구조 및 파일 분할을 적용해 볼 수 있었습니다.
    • (폴더 및 파일 분할이 생각처럼 되지 않았었기 때문에, 강의를 들으면서 정리 해놓았던 파일들을 많이 활용하였습니다.)
    • 분할 작업을 하면서, React(JS, Typescript) 구조에 더욱 친숙해진 느낌을 받았습니다.
    • 다양한 리액트 라이브러리들을 사용해볼 수 있는 기회였습니다.

    ◈ 총 리뷰

    • 강의에서 강조하던 실무에 맞는 폴더구조와 파일구조를 직접 적용해볼 수 있었습니다.
    • React와 더 친해질 수 있었던 프로젝트로, 앞으로 프론트엔드 개발자로 일함에 있어서 적용할 수 있을 것 같다는 생각이 들었습니다.
    • 그리고 앞으로 실무를 통해, 더 많은 것들을 습득하고 여러 기능들을 구현하게 될 미래의 모습을 기대하게 만드는 프로젝트였습니다.
